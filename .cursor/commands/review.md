# Review Command

Review the selected code as a senior Python backend engineer.

## Instructions

Review code according to project standards:

- DRY
- SOLID
- KISS
- PEP8
- Clean Architecture

Verify:

- Correctness
- Maintainability
- Readability
- Performance
- Security
- Architecture boundaries
- Type hints
- Documentation standards

Check for:

- Duplicated logic
- Missing docstrings
- Missing type hints
- Layer violations
- Unnecessary abstractions
- Overengineering
- Potential bugs
- Edge cases

Prioritize findings by severity.

## Output Format

1. **Critical Issues**
2. **Important Improvements**
3. **Minor Suggestions**
4. **Positive Observations**
5. **Overall Assessment**
