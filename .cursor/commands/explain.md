# Explain Command

Analyze and explain the selected code or concept from the prompt.

## Instructions

- Provide clear, technical explanation of the code/concept
- Break down complex logic into understandable parts
- Explain the "why" behind implementation choices
- Highlight potential gotchas or edge cases
- Include relevant context (patterns, best practices, etc.)
- Keep explanations expert-level and concise

## Output Format

1. **Overview**: Brief summary of what the code does
2. **Detailed Breakdown**: Step-by-step explanation of key parts
3. **Key Concepts**: Important patterns, techniques, or principles used
4. **Considerations**: Edge cases, performance, or security notes (if applicable)
